9 razões pelas quais você deve começar a usar classes de dados Python

Em direção à eficiência e menos código clichê

Ahmed Besbes

19 de setembro de 2021 · 7 min de leitura

A partir da versão 3.7, o Python introduziu classes de dados (consulte PEP 557 ), um novo recurso que define classes que contêm e encapsulam dados.

Recentemente, comecei a usar este módulo em alguns projetos de ciência de dados e estou gostando muito. De cabeça, consigo pensar em duas razões:

  1. Menos código clichê
  2. Mais legibilidade e melhor manutenção do código

Este post é um resumo das minhas primeiras impressões: vou usá-lo para apresentar classes de dados e quais problemas eles foram projetados para resolver e passar por 9 recursos interessantes que eles fornecem.
Às vezes, comparo classes escritas com dataclasses com implementações nativas do Python e detecto as diferenças.

Menos conversa, mais código. Vamos dar uma olhada 🔍

PS: Eu não vou cobrir tudo sobre aulas de dadosc mas os recursos pelos quais falaremos devem deixá-lo atualizado. No entanto, se você quiser uma visão geral detalhada, dê uma olhada nos links na seção de recursos no final.

0 — Dataclasses: o panorama geral

Dataclasses, como o nome sugere claramente, são classes que se destinam a armazenar dados. A motivação por trás deste módulo é que às vezes definimos classes que atuam apenas como contêineres de dados e, quando fazemos isso, gastamos uma quantidade consequente de tempo escrevendo código clichê com toneladas de argumentos, um __init__método feio e muitas funções substituídas.

Dataclasses alivia esse problema enquanto fornece métodos úteis adicionais sob o capô. Além disso, como as classes de dados são relativamente novas no ecossistema Python, elas impõem práticas modernas, como anotações de tipo.

👉 dataclasses continuam sendo aulas. Portanto, você pode implementar qualquer método personalizado neles como faria em uma classe normal.

Ok, vamos vê-los em ação agora.

3 — Representações personalizadas dos objetos

Graças ao __repr__método já adicionado pelas classes de dados, as instâncias têm uma representação legal e legível quando são impressas na tela.

Isso facilita a depuração.

Essa representação pode ser substituída para implementar qualquer mensagem personalizada desejada.

4— Fácil conversão para tupla ou dicionário

As instâncias podem ser facilmente serializadas em dicts ou tuplas. Isso é muito útil quando seu código interage com outros programas que esperam esses formatos.

5 — Instâncias congeladas/objetos imutáveis

Usando classes de dados, você pode criar objetos que são somente leitura. Tudo o que você precisa fazer é definir o frozenargumento para Truedentro do @dataclassdecorador.

Ao fazer isso, você impede que qualquer pessoa modifique os valores dos atributos assim que o objeto for instanciado.

Se você tentar definir o atributo de um objeto congelado para um novo valor, um FrozenInstanceErrorerro será gerado.

9 — Comparar objetos e classificá-los

Um recurso útil quando você lida com objetos que contêm dados é a capacidade de compará-los e classificá-los na ordem desejada.

Por padrão, dataclasses implementam __eq__. Para permitir os outros tipos de comparação ( __lt__(menor que), __le__(menor ou igual), __gt__(maior que) e __ge__(maior ou igual)), temos que definir o orderargumento Trueno @dataclassdecorador.

@dataclasses(pedido=Verdadeiro)

A maneira como esses métodos de comparação são implementados pega cada campo definido e os compara na ordem em que são definidos até que haja um valor que não seja igual.

Vamos embrulhar

Dataclasses fornece muitos recursos que permitem que você trabalhe facilmente com classes que atuam como contêineres de dados.

Em particular, este módulo ajuda a:

  • escreva menos código clichê
  • representar objetos em um formato legível
  • implementar pedidos personalizados e comparações
  • acessar rapidamente os atributos e inspecioná-los
  • use métodos especiais como __post_init__ para realizar a inicialização de atributos que dependem de outros
  • definir campos internos...

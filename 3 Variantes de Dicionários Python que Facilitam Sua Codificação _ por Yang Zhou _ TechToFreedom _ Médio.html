3 variantes de dicionários Python que facilitam sua codificação

Escolha o mais adequado

Um dicionário em Python, que contém pares chave-valor, é uma das principais estruturas de dados para programação diária. Além do básico dict, o Python gentilmente nos dá opções mais convenientes. Usá-los habilmente tornará nosso código mais elegante e nossa codificação mais suave.

Este artigo irá mergulhar em 3 variantes comuns de dicionários Python, desde a sintaxe básica até cenários reais. Após a leitura, você poderá escolher o mais adequado em casos reais.

1. Defaultdict: Defina um valor padrão para seu Dict

Quando você obtém o valor de um dict por uma chave, uma exceção inesperada pode ser gerada:

>>> cidade = {'Reino Unido':'Londres','Japão':'Tóquio'} 
>>> print(cidade['Itália'])# KeyError: 'Itália'

Uma solução para evitar o problema acima é usar a get()função para exigir um valor por uma chave:

cidade = {'Reino Unido':'Londres','Japão':'Tóquio'} 
print(city.get('Itália'))# Nenhum

A getfunção pode retornar a Noneem vez de gerar uma exceção quando uma chave não existe.

Mas se nosso programa pode precisar exigir valores por chaves inexistentes muitas vezes. Outra solução é usar o defaultdict:

from collections import defaultdict 
city ​​= defaultdict(str) 
city['UK'] = 'London'print(city['Itália']) 
#

Como mostrado acima, podemos determinar o que deve ser retornado ao definir um arquivo defaultdict. Essa é uma boa ideia para evitar exceções inesperadas.

2. OrderedDict: mantenha a ordem de inserção do seu Dict

Como o próprio nome indica, o OrderedDicté um dict cuja ordem de inserção dos pares de valores-chave será mantida. Será super útil se a ordem for importante.

from collections import OrderedDict 
nums = OrderedDict() 
nums['one'] = 1 
nums['five'] = 5 
nums['dez'] = 10 
print(nums) 
# OrderedDict([('one', 1), ( 'cinco', 5), ('dez', 10)])

Felizmente, se você estiver usando o Python 3.7+, a natureza da preservação da ordem de inserção também se aplica aos dicionários normais:

nums = dict() 
nums['um'] = 1 
nums['cinco'] = 5 
nums['dez'] = 10 
print(nums) 
# {'um': 1, 'cinco': 5, 'dez' : 10}

No entanto, há um ponto propenso a bugs que devemos estar cientes:

from collections import OrderedDict 
nums1 = OrderedDict({'one': 1, 'five': 5, 'ten': 10}) 
nums2 = OrderedDict({'one': 1, 'ten': 10, 'five': 5 }) 
print(nums1 == nums2) 
# False

O código acima compara dois OrderedDictobjetos cujos itens são iguais, mas a ordem de seus itens é diferente. O resultado é False.

E os dicionários normais?

nums1 = {'um': 1, 'cinco': 5, 'dez': 10} 
nums2 = {'um': 1, 'dez': 10, 'cinco': 5} 
print(nums1 == nums2) 
# Verdadeiro

Uma pequena surpresa? O exemplo acima (testado usando Python 3.8) nos disse que dicionários e OrderedDictobjetos normais são diferentes, mesmo nas versões do Python 3.7+.

Simplificando, a ordem dos itens é importante ao comparar OrderedDictobjetos.

3. Contador: uma maneira pítonica de contar itens

Minha variante favorita do Python dict é o Counter. Porque isso pode tornar nosso código Python mais curto e mais elegante.

Por exemplo, se precisarmos contar quantas vezes cada letra é usada em um pedaço de texto, a maneira mais intuitiva pode ser escrever um loop for para percorrer todas as letras e calcular os números.

Mas se você souber Counter, a tarefa acima será tão simples quanto o seguinte código:

from collections import Counter 

title = "3 variantes de dicionários Python que facilitam sua codificação"chars = Contador(título)print(chars) 
# Contador({' ': 9, 'a': 6, 'i': 6, 'o': 5, 'r': 4, 'n': 4, 't': 4, ' s': 3, 'e': 3, 'h': 2, '3': 1, 'V': 1, 'f': 1, 'P': 1, 'y': 1, 'D' : 1, 'c': 1, 'T': 1, 'M': 1, 'k': 1, 'Y': 1, 'u': 1, 'C': 1, 'd': 1 , 'g': 1, 'E': 1})

Como o próprio nome indica, o Counterobjeto nos ajuda a fazer a parte do cálculo e salva os resultados como um dicionário.

Além disso, existe um método muito útil Counterchamado most_common().

Com base no código anterior, se quisermos imprimir as duas primeiras letras que são usadas com mais frequência, o most_common()método pode ajudar:

print(chars.most_common(2)) 
# [(' ', 9), ('a', 6)]

Como mostrado acima, esse método pode listar os principais nitens e suas contagens do mais comum ao menos. (Se nfor Nenhum, ele listará todos os itens e contará do mais comum ao menos.)

Principais conclusões

  1. Use a defaultdictvariante se precisar de valores por chaves inexistentes.
  2. Use a OrderedDictvariante se a ordem dos itens for importante.
  3. Use Countervariante para contar itens convenientemente.
